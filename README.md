2. PowerPuffDB: The Battle Records
Focus: Self-referencing joins, data integrity, and weighted scoring.
Key Features:
- Bloodline Preservation: Managing family ties (siblings/parents) through self-joins in the FamilyRelation table.
- Strength Scoring: Calculating power levels using a weighted formula: (Powers x Rank) + Battles.
- Combat History: Identifying undefeated creatures and tracking the most frequent battle opponents.

3. SPJ: Suppliers-Parts-Jobs-Logistics
Focus: Central Hub (Junction) architecture and supply chain logic
Key Features:
- The Hub Model: A three-way junction table (Shipments) connecting Suppliers, Parts, and Jobs.
- Relational Division: Finding parts shipped by more than one supplier or those never shipped to any job.
- Set Operations: Identifying geographical overlaps where both a supplier and a job are located in the same city.
